## Running it

- First code cell installs deps: `torch torchvision torchaudio`, `torch-geometric`, `pykeen`. Assume a GPU runtime (`device = cuda`).
- Cells are **order-dependent and share state**. Part A section A.3 must run before Part B section B.5.2 because it produces the embeddings file the GNN consumes. The Planetoid dataset is re-loaded in several places (`Planetoid(root='data/PubMed', name='PubMed')`).
- Edit notebook cells with the NotebookEdit tool, not text find/replace.
- The local machine is Windows; the notebook itself targets Colab/Linux. Don't assume generated artifacts exist locally (see below).

## Architecture: one dataset, two representations, one bridge

The whole lab hinges on the **same PubMed graph viewed two ways**, joined by a single file:

1. **KGE view (Part A):** edges are turned into string triples `"paper_{h}", "cites", "paper_{t}"`, wrapped in a PyKEEN `TriplesFactory`, then `.split([0.8, 0.1, 0.1], random_state=2026)`. PyKEEN assigns its **own** internal entity IDs via `entity_to_id` ÔÇö these do **not** match PyG node IDs.
2. **GNN view (Part B):** the raw PyG `data` object with `data.x` (TF-IDF), `data.y` (labels), `data.edge_index`, and `train_mask`/`test_mask`, indexed by PyG node ID `0..19716`.
3. **The bridge ÔÇö `best_entity_embeddings.npy` (shape 19717├Ś128):** the best KGE model (RotatE) embeddings, **re-ordered from PyKEEN IDs into PyG node-ID order** before saving, so B.5.2 can drop them straight into `data.x`.

### The critical gotcha: entity-ID alignment

When saving KGE embeddings for reuse as GNN features, you must convert PyKEEN's entity ordering back to PyG node order. The notebook does this by parsing the integer out of each `"paper_{id}"` label (in the A.3 RotatE cell):

```python
rotate_embs_ordered[int(name.split('_')[1])] = rotate_embs_raw[pykeen_idx]
```

B.5.2 then loads the file positionally (`data.x = torch.tensor(np.load(...))`). If you change how entities are named, split, or saved, this alignment silently breaks and the GNN trains on mismatched features. Keep the `paper_{int}` naming convention intact.

## Conventions that must stay consistent

- **Reproducibility seed `2026` everywhere** ÔÇö `random_state=2026` for splits, `random_seed=2026` for every PyKEEN `pipeline(...)`. This is load-bearing: A.1's "paper A" selection (first paper with 8ÔÇô12 outgoing citations, sorted numerically) is only deterministic because the split is seeded.
- **MRR is the primary selection metric** (`mean_reciprocal_rank`), not Hits@K. Early stopping in A.3 monitors validation MRR: `stopper='early'`, `patience=5`, `relative_delta=0.002`.
- **Shared KGE hyperparameters** across the 5 models: `embedding_dim=128`, `num_epochs=100`, `batch_size=512`. The assignment explicitly wants "sensible" values, not tuned ones ÔÇö don't over-optimize unless a cell is specifically the hyperparameter sweep.
- **GNN architecture** (`GNN` class): stacked `GCNConv` layers Ôćĺ `ReLU` Ôćĺ `Dropout(0.5)` Ôćĺ final `nn.Linear` classifier, `hidden_channels=256`. `forward(x, edge_index, return_embs=False)` returns pre-classifier embeddings when `return_embs=True` ÔÇö this same flag powers B.7. Training: Adam `lr=0.01, weight_decay=5e-4`, `cross_entropy` on `train_mask`, accuracy on `test_mask`.
